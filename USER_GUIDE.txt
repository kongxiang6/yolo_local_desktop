YOLO训练工具 - 操作说明

这份说明面向最终用户编写，尽量不用专业术语。不会 Python、不会命令行，也可以照着做。

一、先认准哪个版本

1. `YOLO训练工具.exe`
   这是当前正式发布的原版稳定入口，建议优先使用。

2. `YOLO训练工具_V2.exe`
   这是并行保留的新版界面预览入口，本次不作为正式发布默认版本。

二、第一次打开怎么做

1. 先完整解压整个压缩包。
2. 不要只单独拿一个 `exe` 运行。
3. 双击打开 `YOLO训练工具.exe`。
4. 第一次打开后，先点“在线一键配置环境”。
5. 等左侧日志提示完成。
6. 再开始标注、训练、预测、跟踪或导出。

三、最稳的使用顺序

推荐按下面这个顺序走：

1. 配置环境
2. 打开标注工作台
3. 整理数据集
4. 开始训练
5. 训练后做验证或预测
6. 最后导出部署模型

四、主界面怎么理解

原版主界面分成几个核心区域：

1. 左侧信息区
   用来看当前任务、日志状态、结果位置和常用操作。

2. 标注工作台
   负责检测框标注、分割标注、自动标注和数据整理。

3. 训练工作台
   负责训练、验证、预测、跟踪。

4. 导出工作台
   负责把 `.pt` 权重导出成 ONNX、TensorRT 等部署格式。

五、标注工作台怎么用

适合做什么：

1. 给图片打检测框
2. 给图片做实例分割
3. 用自动标注先打一遍，再人工修正
4. 把标注整理成训练可直接使用的数据集

推荐流程：

1. 先选择图片目录或项目目录。
2. 设置类别名。
3. 开始手动标注，或者先跑自动标注。
4. 检查类别名和标注结果是否正确。
5. 标注完成后，点击整理数据集或切到训练入口。

六、数据整理怎么用

这个功能用来把原始标注整理成标准训练数据。

支持的常见来源：

1. YOLO TXT
2. LabelMe
3. COCO JSON
4. 自动识别

使用建议：

1. 如果你已经有规范的 `dataset.yaml`，可以直接用于训练。
2. 如果你只有图片和标注文件，先在这里整理。
3. 整理完成后检查生成的训练集、验证集和类别名。

七、训练怎么用

训练前至少准备好三样东西：

1. 数据集入口
   检测、分割、姿态、旋转框一般用 `dataset.yaml`。

2. 模型
   新项目通常选官方预训练模型，继续训练则选本地 `.pt`。

3. 任务类型
   例如检测、分割、分类、姿态、旋转框。

最常用的几个参数：

1. `epochs`
   训练多少轮，新手先从 `50` 或 `100` 开始。

2. `batch`
   一次送多少张图进去，显存不够时先减小它。

3. `imgsz`
   输入尺寸，常见先用 `640`。

4. `device`
   显卡通常填 `0`，只用 CPU 时填 `cpu`。

5. 预设
   拿不准时，优先使用推荐预设。

八、验证、预测、跟踪怎么用

1. 验证
   用来检查训练后的模型效果，重点看指标和结果图。

2. 预测
   用来跑图片、文件夹、视频、摄像头或 RTSP/HTTP 流。

3. 跟踪
   用来做视频或摄像头目标跟踪。

实用建议：

1. `conf` 越高，误检通常越少，但也可能漏检。
2. `iou` 一般先保持默认。
3. `save` 建议保持开启，方便回看结果。

九、导出怎么用

导出工作台用来把训练好的模型转成部署格式。

常见格式：

1. ONNX
2. TensorRT
3. OpenVINO
4. TorchScript
5. TFLite

如果你完全拿不准：

1. 先选 `ONNX`
2. `dynamic` 先保持关闭
3. `half` 和 `int8` 没把握就先别开

十、推荐给新手的默认策略

1. 模型系列优先选最新常用系列。
2. 模型大小优先从 `n` 或 `s` 开始。
3. 参数看不懂就先别改。
4. 先跑通，再慢慢调参。

十一、关于环境配置

“在线一键配置环境”做的事情：

1. 检查软件自己的内置运行环境是否完整
2. 缺少时自动创建
3. 损坏时自动修复
4. 安装训练和推理所需依赖

注意：

1. 这个按钮主要处理软件自己的环境。
2. 不建议手动把环境改到很乱后再继续用。
3. 出现问题时，优先重新点一次“在线一键配置环境”。

十二、关于分辨率和缩放

当前版本已经适配常见分辨率和高 DPI 场景：

1. `720P`
2. `1K / 1600x900`
3. `1080P`
4. `2K`
5. `4K`
6. Windows `100% / 125% / 150% / 200%` 缩放

如果你看到界面还是挤：

1. 先确认系统缩放比例
2. 尽量不要把窗口压得特别小
3. 低分辨率下优先使用默认窗口大小

十三、最常见的问题

1. 为什么不能只发一个 `exe`？
   因为程序还依赖其它资源文件、预设和说明文件。

2. 第一次运行为什么要先配置环境？
   因为软件要先准备自己的 Python 和依赖环境。

3. 显存不够怎么办？
   先减小 `batch`，再减小 `imgsz`。

4. 训练很慢怎么办？
   先看是不是用了 CPU；再检查 `batch`、`imgsz`、`workers`。

5. 自动标注后类别名不对怎么办？
   先回到标注工作台确认类别名，再重新整理或导出数据集。

十四、出错时先看哪里

按下面顺序排查：

1. 看左侧实时日志
2. 点“打开日志”
3. 看结果目录里有没有输出
4. 再决定是重跑任务，还是重新配置环境

十五、给维护者的交付建议

如果你要把软件发给别人：

1. 优先发整个 ZIP 包
2. 提醒对方完整解压
3. 提醒第一次先配置环境
4. 如果是正式交付，优先发原版稳定入口
