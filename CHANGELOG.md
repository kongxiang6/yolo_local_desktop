# Changelog

本项目变更记录参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，并尽量遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [1.1.2] - 2026-05-11

### Changed

- 统一原版与 V2 的完整发布流程，统一改走 `build_exe.ps1 + spec`。
- 默认完整发布包不再附带 `runtime`，降低分发体积与重复拷贝成本。
- 默认完整发布目录不再附带 `presets` 文件夹。
- 更新 `README.md`、`YOLO训练工具操作说明.txt`、`delivery/README_FOR_SHARE.txt` 和本地会话承接文档，确保说明与当前发布规制一致。

### Fixed

- 修复大图切片后自动进入标注时打开了错误目录的问题。
- 修复数据整理时类别名长度与标签最大类 ID 不一致，导致 `dataset.yaml` 元数据无效的问题。
- 修复自动标注回填时覆盖后端返回类别名的问题。
- 修复窗口版 `exe` 调用后台命令时，日志输出触发 `[Errno 22] Invalid argument` 崩溃的问题。
- 修复 V2 训练页和导出页右侧日志预览不实时刷新、切换页面后才显示的问题。
- 修复 V2 训练页与导出页日志区域高度过小的问题。
- 修复发布 ZIP 已被占用时直接失败的问题，打包脚本现在会先尝试改名旧包再继续压缩。

## [1.1.1] - 2026-05-08

### Changed

- V2 顶部导航与主内容区继续按新提案图细化，强调页面自由切换。
- V2 在 `720P` 下继续压缩边距和卡片高度，降低内容被裁切的概率。
- 补充 `_selftest_ui_layout.py` 用于覆盖 V2 多分辨率布局回归。
- 更新 `README.md` 与 `YOLO训练工具操作说明.txt`，同步 UI 与发布方式变化。
- `build_exe.ps1` 开始支持按 spec 生成不同版本的 `dist/release` 目录。

### Fixed

- 修复滚动容器在部分页面切换后失效、滚轮不响应的问题。
- 修复 V2 某些子页面缺少返回主页面按钮的问题。
- 修复 V2 打包误覆盖原版发布目录，导致 V2 `release` 产物混入原版的问题。
- 修复部分 V2 子页面切换时出现明显高度变化的问题。

## [1.1.0] - 2026-05-05

### Added

- 新增并行保留的 V2 界面，用于按整套产品提案图持续重构桌面工作流。
- 接入独立标注工作台，强化标注与训练入口的衔接。
- 开始系统覆盖 `720P / 1K / 1080P / 2K / 4K` 与 `100% / 125% / 150% / 200%` 缩放场景。
- 补充多项 UI 自检与 AI 工具相关自检脚本。

### Changed

- 重构 V2 页面结构，突出训练、导出、标注等主流程入口。
- Windows 高 DPI 场景下同步处理 Tk 缩放和窗口默认尺寸。
- V2 训练页与导出页改为更接近连续流程的视觉结构。
- 原版继续保留稳定交付定位，V2 保持预览迭代定位。
- `build_exe.ps1` 开始用于统一输出 `release` 发布目录。

### Fixed

- 修复标注界面在低分辨率下底部按钮显示不全的问题。
- 修复数据整理时部分类别 ID 解析异常的问题。
- 修复自动标注后类别名被错误补成占位名的问题。
- 修复部分页面滚轮无法使用的问题。
- 修复多个页面在高 DPI 下的 UI 挤压问题。
- 修复页面切换时布局高度不稳定的问题。

## [1.0.2] - 2026-04-28

### Changed

- 运行时改为支持按需准备，`runtime` 不再总是默认打包。
- 补充更多一键配置环境诊断信息。
- 优化环境配置镜像源与自动检测逻辑。

### Fixed

- 修复显卡检测和 Torch 安装方案不一致时的残留问题。
- 修复某些环境下自动重装 Torch 后仍沿用旧状态的问题。
- 修复 `RTX 50 / Blackwell` 等新显卡错误回退 CPU 的问题。
- 修复部分 Torch 版本组合下 CUDA 不可用的问题。
- 修复环境配置失败后的部分恢复逻辑。
- 修复打包后运行时路径识别问题。
