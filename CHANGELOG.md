# Changelog

本项目的重要变更记录在这里。
格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [1.1.0] - 2026-05-05

### Added

- 原版稳定入口接入独立“标注工作台”，整合检测标注、分割标注、自动标注、数据整理和训练回填。
- 新增 V2 并行入口与配套打包脚本，作为独立设计版保留在仓库中。
- 新增多分辨率与高 DPI 自测覆盖，包含 `720P / 1K / 1080P / 2K / 4K` 和 `100% / 125% / 150% / 200%` 缩放。
- 新增标注、分割、AI 工作流、阶段工具和运行时检测相关自测脚本。

### Changed

- 原版和 V2 的默认窗口大小改为按屏幕有效工作区自适应收放，避免首开窗口过大或过小。
- Windows 下启用 DPI 感知与 Tk 缩放同步，改善高缩放场景下的清晰度和尺寸判断。
- V2 在低分辨率下自动切换到紧凑布局，减少侧栏、底栏和重复信息区挤占空间。
- 重新整理仓库文档和操作说明，明确原版为正式发布入口，V2 为并行预览版本。
- `build_exe.ps1` 调整为保留不同版本的 `release` 产物，不再互相清理。

### Fixed

- 修复大图切片完成后打开错误目录，导致无法直接进入标注的问题。
- 修复数据整理时类别名数量可能短于标签实际类 ID，生成错误 `dataset.yaml` 的问题。
- 修复自动标注回填时覆盖后端返回类别名的问题。
- 修复多个页面缺少返回主页面按钮的问题。
- 修复滚轮滚不动、低分辨率布局溢出、页面切换不顺畅等 UI 交互问题。
- 修复完整打包后再次执行另一个版本打包会清理前一个发布目录的问题。

## [1.0.2] - 2026-04-28

### Changed

- 默认发布包改为不内置 `runtime`，只有显式传入 `-IncludeRuntime` 时才一起打包。
- 发布目录中的重复压缩包合并为单一产物，简化交付结构。
- 更新仓库说明和打包说明，统一相对路径写法。

### Fixed

- 运行环境区域简化为固定内置模式，只保留环境检查和一键配置入口。
- 增强依赖安装时的镜像测速、超时与重试逻辑，重复配置优先复用缓存。
- 修复 `RTX 50 / Blackwell` 等新显卡未返回 CUDA 版本时被误判成 CPU 方案的问题。
- 修复已有 Torch 类型与推荐方案不一致时环境残留冲突的问题。
- 修复环境检查遇到损坏依赖时直接失败退出的问题。
- 修复打包脚本中残留的本机私有 `runtime` 搜索路径。

### Added

- 环境检查结果增加更明确的异常摘要。
- 补充运行环境检查自测覆盖。
- 发布产物附带 `LICENSE` 文件。

## [1.0.1] - 2026-04-25

### Changed

- “在线一键配置环境”改为始终只检查和配置软件内置 `runtime`。
- 更新界面提示和分发说明，明确环境配置范围。

### Fixed

- 修复一键配置环境时误把依赖安装到手动选择解释器路径下的问题。

## [1.0.0] - 2026-04-25

### Added

- 新增训练、验证、预测、跟踪、导出和数据整理的一体化桌面工作流。
- 新增运行环境检测、内置环境优先、一键配置环境和环境修复逻辑。
- 新增参数提示、自定义预设、推荐预设、日志查看和结果目录一键打开能力。
- 新增 `USER_GUIDE.txt`、`delivery/README_FOR_SHARE.txt` 和面向 GitHub Release 的发布说明。

### Changed

- 整理仓库文档结构，补充 `README.md` 中的项目说明、发布建议和文档索引。
- 统一辅助文件命名，减少中文文件名在部分 Git 工具中的转义显示问题。
- 固定 GitHub Release 使用的无运行时压缩包命名方式。

### Fixed

- 修复参数预设在折叠区域下可能漏保存、漏加载的问题。
- 修复运行环境选择逻辑误把主程序 `exe` 当成 Python 解释器的问题。
- 修复在线环境配置、滚轮滚动、下拉弹层、日志显示和部分 UI 交互细节问题。
