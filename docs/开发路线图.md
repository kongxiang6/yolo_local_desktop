# 开发路线图

## 项目定位

当前项目已经具备：

- 运行环境检查与在线配置
- 检测数据集整理
- YOLO 训练 / 验证 / 预测 / 跟踪 / 导出
- 参数提示、推荐预设、自定义预设

后续路线不建议继续只在“训练参数页”上无限加细节，而是要把项目逐步升级成：

> **一个面向普通用户的本地 YOLO 数据准备与训练综合工具**

---

## 总体规划原则

### 1. 先做闭环，再做扩展

优先把：

`标注 / 自动标注 → 数据整理 → 训练 → 导出`

这条主链补完整，再考虑更大的模型生态和花哨功能。

### 2. 先做检测，再扩到分割 / 姿态 / OBB

因为检测任务最常用，也最适合作为第一版标注器落地。

### 3. 先做本地可用，再做大而全

优先保证：

- 新手能看懂
- 普通电脑能跑
- 功能能打通
- 出错能定位

---

# 第一期：补齐“标注 → 训练”基础闭环

## 目标

把当前项目从“训练工具”升级成“基础数据准备 + 训练工具”。

## 核心功能

### 1. 标注工作台（检测版）

- 新增“标注”页签
- 支持打开单张图片、文件夹图片
- 支持框选目标框
- 支持编辑 / 删除目标框
- 支持类别管理
- 支持上一张 / 下一张切换
- 支持保存为 YOLO 检测标注

### 2. YOLO 自动标注（检测版）

- 选择本地检测模型
- 对当前图片或整批图片一键自动打框
- 自动生成初始标注结果
- 支持设置基础阈值：`conf / iou / imgsz`

### 3. 自动标注结果人工修订

- 自动框生成后允许人工拖动、删除、补框
- 支持重新指定类别
- 支持保存修订结果

### 4. 标注结果直接进入训练

- 标注完成后可一键导出为标准检测数据集
- 自动生成 `images/train`、`images/val`、`labels/train`、`labels/val`、`dataset.yaml`
- 可直接回填到当前训练入口

### 5. 基础数据集项目管理

- 记录当前标注目录
- 记录类别列表
- 记录最后打开位置
- 支持恢复上次标注状态

## 交付结果

- 新增 `标注` 主功能页
- 支持检测框手工标注
- 支持检测模型自动标注
- 支持人工修订后导出训练集
- 与现有训练流程打通

## 这一期先不做

- 分割标注
- 姿态标注
- OBB 标注
- 视频标注
- OCR
- 文本提示标注

## 验收标准

- 普通用户不需要命令行就能完成一套检测数据标注
- 标注结果能直接进入当前训练流程
- 自动标注后的框能在界面中正常编辑
- 导出的数据集能被当前训练入口直接识别

## 优先级

- 最高

---

# 第二期：扩展到“交互式分割 + 视频标注 + 多任务数据整理”

## 目标

把工具从“检测数据闭环”扩展成“多任务数据准备工具”。

## 核心功能

### 1. SAM / SAM2 交互式分割

- 点选分割
- 框选分割
- 掩码预览
- 掩码修订
- 掩码导出

### 2. 视频跟踪辅助标注

- 打开视频文件或摄像头视频流
- 基于跟踪结果沿用目标 ID
- 减少逐帧重复打框
- 支持关键帧修正

### 3. 分割 / 姿态 / OBB 数据整理向导

- 分割数据集整理
- 姿态数据集整理
- OBB 数据集整理
- 分类数据集整理

### 4. 多任务自动标注扩展

- 分割自动标注
- 姿态自动标注
- OBB 自动标注

### 5. 数据集质量检查

- 类别缺失提醒
- 空标注提醒
- 图片 / 标签不匹配提醒
- 类别 ID 越界提醒

## 交付结果

- 支持交互式分割
- 支持视频辅助标注
- 支持多任务数据集整理
- 支持检测之外的自动标注能力

## 这一期先不做

- OCR / 文档版面分析
- 文本提示检测
- 文本提示分割
- 大图切片自动标注
- 模型中心 / 模型商店

## 验收标准

- 用户可以完成检测与分割两类主要数据准备流程
- 视频跟踪标注能明显减少重复操作
- 新整理出的多任务数据集能接入对应训练流程

## 优先级

- 高

---

# 第三期：做“综合 AI 标注平台”扩展

## 目标

把项目从“本地训练工具 + 数据准备工具”继续升级成“多模型自动标注综合平台”。

## 核心功能

### 1. 文本提示检测 / 分割

- 输入文字描述自动找目标
- 文本找目标后直接生成掩码
- 支持 `GroundingDINO / Florence2 + SAM` 类能力

### 2. OCR / 文档 / 车牌专项能力

- 文本检测
- 文本识别
- 车牌框选 / 识别
- 文档布局分析

### 3. SAHI 大图切片自动标注

- 大图自动切片
- 小目标密集场景推理
- 结果自动合并与去重

### 4. 模型管理器

- 模型列表
- 模型分类筛选
- 模型下载
- 模型缓存管理
- 模型依赖缺失提示

### 5. 可扩展模型架构

- 后续接入更多自动标注后端
- 统一模型调用接口
- 统一结果回写接口

## 交付结果

- 多模型自动标注平台雏形完成
- 文本提示、OCR、大图切片等高级能力可用
- 模型扩展不再依赖手工硬编码

## 这一期的定位

- 更偏“平台化”
- 更偏“高级用户 + 专业场景”
- 更适合做成长期演进方向

## 验收标准

- 用户可以根据不同场景切换不同自动标注模型
- 大图、小目标、文本类场景有专门能力入口
- 新模型接入成本明显下降

## 优先级

- 中高

---

## 建议执行顺序

### 第一优先

- 第一期全部做完

原因：

- 最贴近当前项目定位
- 与现有训练链最容易打通
- 最容易尽快产生实际价值

### 第二优先

- 第二期中的：
  - SAM 交互分割
  - 视频跟踪辅助标注
  - 多任务数据整理

原因：

- 这些功能和现有 YOLO 训练用户需求最接近

### 第三优先

- 第三期中的：
  - 文本提示标注
  - OCR
  - SAHI
  - 模型管理器

原因：

- 价值高，但复杂度和维护成本也更高

---

## 推荐里程碑

### 里程碑 1

- 完成检测标注工作台
- 完成检测自动标注
- 完成导出标准训练集

### 里程碑 2

- 完成交互式分割
- 完成视频跟踪标注
- 完成多任务数据整理

### 里程碑 3

- 完成文本提示标注
- 完成 OCR / 文档专项
- 完成模型管理器

---

## 最终建议

如果只选一条最稳的路线：

> **先用第一期把“训练工具”升级成“检测数据准备 + 训练闭环工具”，再进入第二期。**

这是当前最适合项目状态、实现成本、用户价值三者平衡的方案。
