# 功能对比表

## 对比对象

- 对方软件：`C:\Users\27468\Desktop\阿斯Yolo综合工具`
- 我们项目：`I:\AI\yolo_local_desktop`

## 结论摘要

- `阿斯Yolo综合工具` 的 **训练 / 验证 / 预测 / 跟踪 / 导出 / 检测数据整理** 这条主链，和我们项目属于同一类方案，甚至后端结构也很接近。
- 对方软件真正多出来的核心，不是在训练器本身，而是在 **标注器 + 自动标注器 + 多模型推理平台**。
- 我们项目当前已经是一个比较完整的 **YOLO 训练工具**，但还不是一个完整的 **“标注 + 自动标注 + 训练闭环平台”**。
- 如果后续要补齐差距，最值得优先做的是：**标注工作台、YOLO 自动标注、SAM/SAM2 交互分割、视频跟踪标注、OCR / 文本提示标注**。

## 三、我们项目当前明确还缺哪些功能

## 高优先级缺项

这些是最值得补的，补完后产品定位会从“训练工具”升级成“数据准备 + 训练闭环工具”。

| 功能 | 当前状态 | 建议 |
|---|---|---|
| 标注工作台 | 缺失 | 建议单独做一个“标注”页签 |
| YOLO 自动标注 | 缺失 | 先从检测框自动标注开始，成本最低 |
| SAM / SAM2 交互分割 | 缺失 | 建议作为第二阶段能力 |
| 视频跟踪辅助标注 | 缺失 | 可复用现有跟踪结果，但要补标注编辑 UI |
| 文本提示检测 / 分割 | 缺失 | 对新手和快速打标很有价值 |

## 中优先级缺项

| 功能 | 当前状态 | 建议 |
|---|---|---|
| OCR / 文档 / 车牌专项模型 | 缺失 | 适合做插件式扩展，不必一口气做全 |
| SAHI 大图切片标注 | 缺失 | 适合遥感、小目标、密集目标场景 |
| 分割 / 姿态 / OBB 数据整理 | 缺失 | 现有数据整理器可继续扩展 |
| 模型管理器 | 缺失 | 后续自动标注模型多了就必须有 |

## 低优先级缺项

| 功能 | 当前状态 | 建议 |
|---|---|---|
| 聊天 / VQA / 远程服务 | 缺失 | 不建议现在优先做 |
| 大而全的模型超市 | 缺失 | 先聚焦主流模型，不要过度铺开 |

## 四、建议开发顺序

### 方案 A：最实用、最稳妥

1. 标注工作台
2. YOLO 检测自动标注
3. 自动标注结果可人工修订
4. 标注结果导出为标准 YOLO 数据集
5. 与现有训练入口打通

适合目标：

- 先把“训练工具”升级成“数据准备 + 训练工具”
- 开发量可控
- 最容易尽快落地

### 方案 B：继续拉开差距

在方案 A 完成后，再继续：

1. SAM / SAM2 交互分割
2. 视频跟踪辅助标注
3. SAHI 大图切片
4. OCR / 文档专项能力
5. 文本提示检测 / 分割

适合目标：

- 做成更完整的综合工具
- 面向更复杂的数据标注场景

## 五、关键判断

- 对方软件的 **训练链条** 不构成我们当前最大的功能落差。
- 最大差距在于：对方已经覆盖了 **“标注 → 自动标注 → 数据整理 → 训练 → 导出”** 的完整闭环。
- 我们现在已经把 **“环境 → 数据整理（检测）→ 训练 → 验证 → 预测 → 跟踪 → 导出”** 这条链做完整了。
- 下一步如果要继续补，最值得做的是 **补前半段：标注与自动标注**，而不是继续在训练参数页里做无上限扩充。

---

## 六、最终建议

如果目标是“给很多普通用户直接用”，建议我们后续优先做下面 5 项：

1. 标注工作台
2. YOLO 自动标注
3. 自动标注结果人工修订
4. SAM / SAM2 交互分割
5. 视频跟踪辅助标注

做完这 5 项之后，我们项目的产品定位会非常清晰：

> **一个面向普通用户的本地 YOLO 数据准备与训练综合工具**
